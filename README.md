# AML-EDGE-Project-health-insurance

1. Technical Workflow (Steps 1–4)The core development process is divided into four distinct technical stages:Step 1: Data Loading & EDALoad the insurance dataset (CSV).Visualize the distribution of charges (identify right-skewness).Analyze relationships using boxplots (smoker, region, sex) and scatter plots (age, BMI).
2. PreprocessingEncoding: Binary encoding for smoker/sex; One-hot encoding for region.Transformation: Apply Log-transform to the target variable (charges) and StandardScaler to features.Step 3: Model TrainingImplement three models: Linear Regression, Ridge (L2), and Random Forest.Validate performance using 5-fold Cross-Validation.Step 4: EvaluationMetrics: MAE, RMSE, and $R^2$.Visuals: Actual vs. Predicted scatter plots and Random Forest Feature Importance.2. GUI Requirements (Step 5)Every project must include a functional interface (Streamlit, Tkinter, or Gradio) with the following:Input Interface: Sliders and dropdowns for patient attributes.Output: Estimated premium and a Confidence Score.Visualization: A bar chart showing the top 3 cost-driving factors.Legal: A visible disclaimer or limitation note.
